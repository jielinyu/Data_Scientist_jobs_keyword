def web_scrape(search_position, search_location, output_path):
    """
    Given desired position, location, and output file path, save the job posting information as a csv file. 
    
    Keyword arguments:
    search_position -- (str) the job position 
    search_location -- (str) the job location
    output_path -- (str) the output file path
    """
    job_list = []
    title = []
    location = []
    company = []
    description = []
    url_http = "https://www.indeed.ca/m/"
    # Scrape 200 jobs 
    for i in range(20):
        time.sleep(1)
        jobs_html = requests.get("https://www.indeed.ca/m/"+"jobs?q="\
                                 +search_position+"&l="+search_location+"&start="+str(i*10))
        job_bs = BeautifulSoup(jobs_html.text)
        for link in job_bs.find_all('a',{'rel':"nofollow"}):
            job_list.append(url_http + link.get('href'))
    # Scrape jobs' information       
    for job in job_list:
        job_50 = requests.get(job)
        bs_50 = BeautifulSoup(job_50.text)
        title.append(bs_50.find('font',{'size':'+1'}).text)
        location.append(bs_50.find('span',{'class':'location'}).text)
        company.append(bs_50.find('span', {'class':"location"}).previous_sibling)
        description.append(bs_50.find('div',{'id':'desc'}).text)
    # Make a dataframe 
    table = pd.DataFrame({'Company':company,
              'Location':location,
              'Title':title,
              'Description':description,
              'Link':job_list})
    # Save csv
    table.to_csv(output_path)

nlp = spacy.load("en")

def preprocess(text,  
               irrelevant_pos = ['ADV','PRON','CCONJ','PUNCT','PART','DET','ADP','SPACE','VERB']): 
    """
    Given text, and irrelevant_pos carry out preprocessing of the text 
    and return a preprocessed string. 
    
    Keyword arguments:
    text -- (str) the text to be preprocessed
    irrelevant_pos -- (list) a list of irrelevant pos tags
    
    Returns: (str) the preprocessed text
    """
    # Remove distracting characters
    text = re.sub(r'\s+', ' ', text)
    text = re.sub(r'''[\*\~]+''', "", text)
    
    # Customized Stopwords 
    # The job postings contains some French as these are jobs in Canada
    stopwords = [
    "jobs", "apply", "email", "password", "new", "please", "required",
    "opportunities", "skills", "sign", "talent", "follow",  "job", "work",
    "click", "home", "next", "us", "start", "take", "enter", "date", "people",
    "career", "hours", "week", "type", "attach", "resume", "search", "advanced",
    "privacy", "terms", "find", "years", "experience", "find", "title", "keywords",
    "company", "indeed", "help", "centre", "cookies", "post", "(free)", "industry",
    "employment", "ago", "save", "forums", "browse", "employer", "browse", "city",
    "province", "title", "api", "review", "application", "help", "instructions", 
    "applying", "full", "time", "opportunity", "team", "join", "resumes", "employers",
    "rights", "reserved", "create", "north", "america", "toronto", "interview", "cover",
    "letter", "30+", "days", "inc", "trends", 'de', 'le', 'ericsson', 'dans', 'des', 
    'working', 'etc.', 'canada', 'les', 'employees', 'accommodation',
    'et', 'montréal','en','mi','yoga','à','équipe','vancuover','se','se se' 
    ]
    
    text = " ".join([i.lower() for i in text.split() if i.lower() not in stopwords])
    
    doc = nlp(text)
    clean_text = []
    
    for token in doc:
        if (token.is_stop == False # Check if it's not a stopword
            and token.is_alpha # Check if it's an alphanumerics char 
            and token.pos_ not in irrelevant_pos): # Check if it's irrelevant pos
            clean_text.append(str(token))
    return " ".join(clean_text)

Method 2: TF-IDF Vectorizer¶

The TF-IDF algorithm is used to weigh a keyword in any content and assign the importance to that keyword based on the number of times it appears in the document. More importantly, it checks how relevant the keyword is throughout the web, which is referred to as corpus.

  • TF refers to term frequency of a term in a document.
  • DF refers to the number of documents of the corpus in which this term occurs.

Clustering - Kmeans¶

I think Kmeans would be the best clustering algorithm to be used in this case. Since we do not have labelled data, I can only use unsupervised learning techniques such as Kmeans, DBScan, hierarchical clustering, and LDA model, etc. Kmeans is one of the most effective clustering algorithm for text data. Kmeans algorithm is an iterative algorithm that tries to partition the dataset into K pre-defined distinct non-overlapping subgroups (clusters) where each data point belongs to only one group. It tries to make the inter-cluster data points as similar as possible while also keeping the clusters as different (far) as possible.

To choose the number of clusters, I use Elbow Method. Elbow method gives us an idea on what a good k number of clusters would be based on the sum of squared distance (SSE) between data points and their assigned clusters’ centroids. We pick k at the spot where SSE starts to flatten out and forming an elbow.

Synopsis¶

  • In summary, this application can serve different groups in society to perform better in the labor market. For displaced workers or employee who are pursuing a different filed of career, they can use this analyzer to help them find in-demand online classes and learning materials to meet jobs' requirements. For example, by using the analyzer, a non-technical employee who wants to transfer his/her career to data science will know that he/she needs to acquire machine learning skills, statistical analysis, computer science and great communication skills in order to be a qualified candidate. In addition, it can also help academic institutions to improve their course design. For example, master of data science program at UBC can use it to find out how course contents they need to modify. The program has already provided machine learning, python, and data visualization classes. By using this analyzer, they may want to design more classes focusing on technical communication and big data analysis to help their students find their dream jobs. Moreover, talent acquisition partners can find the right candidates with right skills for their company easier.

Web Scraping Challenges¶

  • Indeed does not provide an API tool for users to analyze data, thus I need to scrape job postings page by page using Python package such as BeautifulSoup.
  • When retrieving key words of job postings from the website version of Indeed, it's hard to find the pattern of each job post url because of its complex html format and there are too many irrelevant information like advertisemnts in each website. Therefore, in order to find important information from each job description easily, I extracted information using the mobile version because it gives me a much cleaner html format.
